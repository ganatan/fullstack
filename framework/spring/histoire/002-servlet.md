# Servlet

1. **1996–1997** : Sun Microsystems conçoit les Servlets  
   Objectif : remplacer CGI par un composant Java persistant

2. **12/1997** : Servlet API **1.0**  
   Naissance du standard web Java côté serveur

3. **1998–1999** : Servlet API **2.0 / 2.1**  
   Introduction de `HttpServlet`, support HTTP natif

4. **2001** : Servlet API **2.3**  
   Intégration complète à J2EE, `web.xml` obligatoire

5. **2005** : Servlet API **2.5**  
   Standard dominant des SI web Java

6. **12/2009** : Servlet API **3.0**  
   Annotations (`@WebServlet`), async, fin du tout XML

7. **2013** : Servlet API **3.1**  
   Non-blocking I/O, base du réactif Servlet

8. **2017** : Servlet API **4.0**  
   Support HTTP/2

9. **2018** : Transfert Java EE vers Eclipse Foundation  
   Début du passage `javax.*` → `jakarta.*`

10. **2020** : Servlet API **5.0** (Jakarta EE 9)  
    Rupture de namespace

11. **2022** : Servlet API **6.0** (Jakarta EE 10)  
    Compatible Java 17+, conteneurs modernes

12. Aujourd’hui :  
    Servlet = fondation du web Java  
    Spring MVC = abstraction au-dessus de Servlet  
    Tomcat / Jetty = conteneurs d’exécution
